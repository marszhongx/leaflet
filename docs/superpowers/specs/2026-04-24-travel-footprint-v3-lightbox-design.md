# 旅游足迹第三版设计文档

## 目标

在第二版 popup 多图详情的基础上，为地点图片增加全屏遮罩预览能力。

第三版仍然保持地图页面结构不变，不引入侧边栏或独立详情页。升级点集中在图片浏览体验：用户点击 popup 中的缩略图后，可以进入全屏大图预览，并在当前地点的图片集合内进行上一张和下一张切换。

## 范围

本次只实现以下内容：

- 保持全屏地图布局不变
- 保持第二版 popup 缩略图展示不变
- 点击 popup 缩略图后打开全屏遮罩预览层
- 在预览层中显示当前点击的大图
- 支持在当前地点的 `images` 数组内进行上一张和下一张切换
- 支持关闭预览并返回地图当前状态
- 将预览层逻辑拆分为独立模块
- 为预览层状态逻辑增加自动化测试

本次不包含以下内容：

- 键盘方向键切换
- 灯箱动画或复杂过渡效果
- 跨地点切换图片
- 独立图片详情页
- 侧边栏相册模式
- 图片缩放、拖拽、手势支持
- 图片加载失败占位图

## 技术选择

### 方案对比

1. 方案 A：popup 缩略图点击进入全屏遮罩预览，并支持上一张/下一张切换
2. 方案 B：popup 缩略图点击进入单图预览，不支持切换
3. 方案 C：全屏遮罩预览 + 上一张/下一张 + 底部缩略图带

### 选定方案

采用方案 A。

原因：

- 最符合当前需求：点击缩略图进入全屏大图，并支持切换
- 能在现有第二版结构上自然扩展，不需要重构页面布局
- 只增加一层预览状态管理，复杂度仍然可控
- 避免底部缩略图带等超过第三版范围的相册化设计

## 页面结构

页面整体结构继续保持为地图主导：

- 地图仍占满主视口
- popup 仍用于展示地点文字和缩略图
- 页面上新增一个按需显示的全屏遮罩预览层

这个预览层不改变地图布局，只在用户点击缩略图时临时覆盖页面。

## 文件划分

建议第三版按以下职责边界扩展：

### `src/popupContent.js`

- 继续负责输出 popup HTML
- 为每张缩略图提供可识别信息，例如图片索引
- 保持只负责 popup 结构，不负责预览层状态

### `src/lightbox.js`

- 新增独立模块
- 负责：
  - 创建全屏预览层
  - 打开预览
  - 关闭预览
  - 切换上一张/下一张
  - 根据当前索引更新显示内容
- 只处理预览层本身，不直接依赖地图初始化逻辑

### `src/main.js`

- 继续负责地图初始化与 marker popup 绑定
- 新增一层事件连接逻辑：
  - 监听 popup 缩略图点击
  - 识别当前地点与当前图片索引
  - 调用 `lightbox.js` 打开对应大图

### `src/places.js`

- 继续作为地点数据源
- 数据结构不需要进一步修改
- `images` 数组同时服务 popup 和预览层

### `src/style.css`

- 保留现有地图与 popup 样式
- 新增：
  - 全屏遮罩层样式
  - 大图容器样式
  - 关闭按钮样式
  - 上一张/下一张按钮样式
  - 单图场景下的按钮隐藏逻辑样式

### `src/lightbox.test.js`

- 为预览层状态逻辑提供测试
- 验证打开、关闭、切换和单图边界行为

## 状态流

第三版建议把预览状态集中在一个很小的模型里，只维护：

- 当前是否打开预览
- 当前地点的图片数组
- 当前图片索引

状态流如下：

1. 用户点击某个 marker，打开 popup
2. 用户点击 popup 中的某张缩略图
3. `src/main.js` 捕获点击事件
4. `src/main.js` 将当前地点的图片数组与点击索引传给 `src/lightbox.js`
5. `src/lightbox.js` 打开预览层并显示对应大图
6. 用户点击上一张或下一张时，只在当前地点图片数组内部切换
7. 用户点击关闭按钮后，预览层关闭并返回地图当前状态

这个状态流保证：

- popup 负责入口
- `main.js` 负责连接
- `lightbox.js` 负责预览层状态与显示

## 交互行为

第三版建议定义如下交互：

- 点击 popup 缩略图后打开全屏遮罩预览层
- 默认显示被点击的那一张大图
- 预览层显示：
  - 当前大图
  - 关闭按钮
  - 上一张按钮
  - 下一张按钮
- 切换只发生在当前地点的 `images` 数组内部
- 关闭预览后回到地图当前状态
- 地图位置不重置，popup 也不因关闭预览而被强制重建

## 边界场景处理

### 只有一张图片

- 仍可进入大图预览
- 不显示上一张和下一张按钮
- 预览层只负责查看和关闭

### 没有图片

- 第二版已经不会渲染图片区
- 因此第三版不会出现可点击的大图预览入口
- 不需要额外逻辑分支

### 图片索引越界

- `lightbox.js` 内部必须保证索引始终落在合法范围内
- 不允许出现空白大图或无效状态

### 关闭后再次打开

- 每次打开都以本次点击的缩略图索引为准
- 不沿用上一次打开时的索引状态

### 图片尺寸不一致

- 通过样式统一大图展示容器
- 图片在视口内按统一规则适配，防止撑坏遮罩布局

## 样式策略

第三版预览层样式目标是清晰、聚焦、轻量：

- 遮罩层覆盖整个页面并弱化背景地图
- 大图区域居中显示
- 关闭按钮位于稳定且易于发现的位置
- 上一张/下一张按钮对称分布于图片两侧或左右区域
- 不引入复杂动画系统
- 单图场景通过按钮隐藏保持界面简洁

## 验收标准

第三版完成时应满足：

1. 点击 popup 缩略图能打开全屏遮罩预览
2. 默认显示被点击的那一张大图
3. 多图地点支持上一张/下一张切换
4. 切换只发生在当前地点的 `images` 数组内
5. 单图地点可打开预览，但不显示切换按钮
6. 关闭预览后返回地图当前状态，不重置地图位置
7. `npm test` 和 `npm run build` 均通过

## 测试策略

第三版采用“预览层状态自动化测试 + 浏览器手动验证”的组合：

### 自动化测试

新增 `src/lightbox.test.js`，覆盖：

- 打开预览时能记录当前图片数组和索引
- 上一张/下一张切换逻辑正确
- 单图场景不会错误切换
- 关闭后状态重置正常
- 如果预览层 DOM 也由该模块创建，则可进一步验证单图时按钮隐藏、多图时按钮显示

### 手动验证

- 启动 dev server
- 打开页面并点击北京或上海 popup 中的缩略图
- 检查是否进入全屏大图预览
- 检查上一张/下一张切换是否正确
- 点击杭州单图缩略图，确认能打开预览且不显示切换按钮
- 关闭预览后确认地图仍保持原状态
- 再次执行 `npm test` 和 `npm run build` 做最终回归验证

## 后续扩展边界

本设计为未来扩展保留清晰边界：

- 若需要键盘方向键切换，可在 `lightbox.js` 上继续增强输入处理
- 若需要底部缩略图带，可在预览层模块内部继续扩展，不影响地图结构
- 若需要跨地点相册模式，则需要重新设计状态来源和切换范围，不属于当前第三版范围

第三版仍坚持 YAGNI，只解决“从 popup 缩略图进入全屏预览，并在当前地点内切图”这一层目标。
