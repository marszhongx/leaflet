# 旅游足迹设计文档

## 目标

构建一个基于 Leaflet 的最小可用旅游足迹页面。

第一版聚焦于最核心能力：打开页面后展示全屏地图，并在地图上显示用户去过的地点。地点数据先保存在前端本地代码中，用户可以通过点击地图上的标记点查看地点名称。

## 范围

本次只实现以下内容：

- 使用 Vite 创建前端项目
- 使用 Leaflet 渲染地图
- 页面默认采用全屏地图布局
- 旅游地点数据先写死在本地 JavaScript 文件中
- 地图上展示所有地点 marker
- 点击 marker 显示地点名称 popup
- 页面初始化时自动缩放到能展示全部地点
- 处理单点场景和空数据场景

本次不包含以下内容：

- 地点在线编辑
- 后端服务或数据库
- 登录、同步、分享
- 路线连线
- 侧边栏地点列表
- 时间轴、筛选、分类等扩展功能

## 技术选择

### 方案对比

1. 方案 A：Vite + Leaflet + 本地静态点位 + 全屏地图
2. 方案 B：Vite + Leaflet + 本地静态点位 + 自动适配视野
3. 方案 C：Vite + Leaflet + 本地静态点位 + 更多交互增强

### 选定方案

采用方案 B。

原因：

- 保持实现最小化，适合当前空项目快速起步
- 比手动设置固定中心点更适合旅游足迹场景
- 随着地点增加，不需要反复手调初始地图视角

## 页面结构

页面采用单屏地图布局。

- 页面主体是一个占满视口的地图容器
- 用户进入页面后直接看到地图
- 不引入额外头部、列表面板或复杂布局

该结构的目标是让第一版专注于“地图展示旅游足迹”本身。

## 文件划分

建议将实现拆分为以下几个最小职责模块：

### `index.html`

- 提供页面基础结构
- 提供地图挂载容器

### `src/main.js`

- 作为应用入口
- 初始化 Leaflet 地图
- 加载底图图层
- 读取地点数据
- 创建 marker 与 popup
- 计算并应用地图初始视野

### `src/places.js`

- 保存本地旅游地点数组
- 每个地点至少包含名称、纬度、经度
- 作为未来扩展为 JSON 或远程数据源的替换边界

### `src/style.css`

- 负责页面全屏布局
- 确保地图容器具备正确高度
- 承担页面最基础的显示样式

## 数据模型

第一版地点数据使用前端本地数组表示。建议结构如下：

```js
[
  { name: '北京', lat: 39.9042, lng: 116.4074 },
  { name: '上海', lat: 31.2304, lng: 121.4737 }
]
```

字段说明：

- `name`：地点名称，用于 popup 展示
- `lat`：纬度
- `lng`：经度

第一版不加入日期、描述、照片、国家、省份等额外字段，避免超出最小范围。

## 数据流

页面启动后的数据流如下：

1. 入口文件创建地图实例
2. 加载底图瓦片图层
3. 从 `src/places.js` 读取地点数组
4. 遍历地点数组并创建 marker
5. 为每个 marker 绑定 popup，显示地点名称
6. 收集全部点位坐标并计算地图边界
7. 调整地图视野，使所有地点都能显示在当前视口中

这个数据流保持单向且简单，便于后续扩展。

## 交互行为

第一版仅保留最必要交互：

- 页面加载即显示地图
- 地图上展示全部旅游地点 marker
- 点击 marker 弹出地点名称
- 地图支持 Leaflet 默认的拖拽和缩放交互

不在本次范围内加入表单输入、地点编辑或复杂控制面板。

## 边界场景处理

### 多个地点

- 使用所有点位计算边界
- 自动缩放和居中，保证用户打开页面后即可看到完整足迹范围

### 单个地点

- 不直接依赖 `fitBounds` 的默认体验
- 对单点采用固定且合理的默认缩放级别，以获得自然展示效果

### 空地点数据

- 页面仍应正常初始化地图
- 地图显示一个默认中心点和默认缩放级别
- 不因为数据为空而报错或出现空白页面

## 验收标准

满足以下条件即可认为第一版完成：

1. 开发服务器可以正常启动
2. 浏览器中可正常显示 Leaflet 地图
3. 本地定义的每个地点都能显示为 marker
4. 点击 marker 能看到地点名称 popup
5. 初始加载时地图会自动展示全部地点
6. 单点场景下地图能正常聚焦
7. 空数据场景下页面不报错且地图可正常显示

## 测试策略

第一版以手动验证为主。

验证步骤：

- 启动本地开发服务器
- 在浏览器打开页面
- 检查地图是否正常渲染
- 检查多个地点时是否全部显示
- 点击 marker 检查 popup 内容
- 将地点数据改为单个地点，检查聚焦效果
- 将地点数据改为空数组，检查页面是否仍能正常显示

当前阶段不要求引入自动化测试框架，因为目标是先完成一个最小可运行版本。

## 后续扩展边界

本设计为后续扩展保留了明确边界：

- 若增加字段，可在 `src/places.js` 扩展数据结构
- 若切换到 JSON 或接口数据，可替换地点数据来源而不必推翻地图渲染逻辑
- 若增加列表面板、路线连线、时间轴等功能，可在现有单页地图基础上继续扩展

当前版本以最小实现为优先，不预先引入未来功能的结构复杂度。
